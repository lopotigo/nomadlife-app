Astro Developer for Pixel-Accurate Landing Page + Stripe + Calendly Integration

Posted yesterday

Worldwide
Summary
Hi, I am looking for a reliable student or junior developer who can build a clean, modern, and responsive website in Astro.

This is not a large agency project. I am looking for someone technically solid, and easy to communicate with.

Scope of work
- The first phase includes:
Build a pixel-perfect Astro website based on my design reference / Figma / existing examples
Make the site responsive for desktop and mobile
- Set up a clean structure so the project remains easy to maintain
- Integrate Stripe Checkout for payments
Integrate Calendly (or open source variant) for booking calls
- Deploy the website so it is live and working
- Provide a short handover documentation with the most important steps

Important
At this stage, I am not looking for a full Moodle or n8n setup in this project.

This first job is focused on:
- Astro frontend
- deployment
- Stripe integration
- Calendly integration (or open source variant)

If the collaboration goes well, there may be a second phase later for more technical platform work.

What I am looking for
Please apply only if you have experience with:
- Astro
- responsive frontend development
- clean HTML/CSS/JavaScript or TypeScript
- integrating Stripe Checkout
- embedding or integrating Calendly
- deployment of modern frontend projects

Nice to have
It is a plus if you also understand:
- DNS / domains / subdomains
- basic server deployment
- Git / GitHub
- working with clear design references

Budget
I prefer working with a fixed-price milestone.
This is intended as a small first project, so please keep your offer realistic and student-friendly.

To apply, please send:
2–3 relevant examples of websites you built yourself
A short explanation of your Astro experience
Whether you have integrated Stripe Checkout before
Whether you have integrated Calendly before
Your proposed fixed price for this first phase
Your estimated timeline

Small test question
Please answer this question in your application:
How would you structure an Astro project so that it stays clean, responsive, and easy to extend later with additional pages or integrations?

I am looking for someone practical, honest, and structured.
Clear communication is very important to me.

Thank you.
